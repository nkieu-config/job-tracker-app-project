# Screenshots for the README

Capture these from the **demo account** (sign in → “Try the demo account”, which
is pre-populated by `npm run seed`). Save each file with the exact name below so
the README image links resolve.

| File | What to show |
| --- | --- |
| `dashboard.png` | `/dashboard` — the status count cards (Saved → Rejected) and the “Upcoming deadlines” list. |
| `jd-analysis.png` | An application detail page (e.g. **Acme Corp · Senior Backend Engineer**) showing the **Skills analysis** section with required skills tagged ✓ matched / ✗ missing. |
| `resume-fit.png` | The **Resume fit** section showing resume versions ranked by match % (run *Compute resume fit* first — needs `GEMINI_API_KEY`). |
| `tailor.gif` | The **Tailor resume bullets** box while text streams in live (a short GIF is best). |

## Tips

- Use a clean browser window (no extensions bar); light **or** dark mode — pick one and be consistent.
- Target ~1400px wide for crisp rendering at the README’s 820/400px display sizes.
- macOS: `Cmd+Shift+5` for screenshots and screen recordings; convert the recording to GIF (e.g. with [Gifski](https://gif.ski/)).
- Keep the streaming GIF short (5–8s) and loop it.
